コードレビューとは

例題 1

バグのあるコードを見つける（正しさ）

Python
# --- バグのあるコード ---
def find_max_buggy(numbers):
    """リストの最大値を返す関数（バグあり）"""
    maximum = numbers[0]  # 空リストでエラー！
    for i in range(len(numbers)):
        if numbers[i] > maximum:
            maximum = numbers[i]
    return maximum

# --- 修正版 ---
def find_max_fixed(numbers):
    """リストの最大値を返す関数（修正版）"""
    if not numbers:  # 空リストチェックを追加
        print("警告: 空のリストです")
        return None
    maximum = numbers[0]
    for i in range(1, len(numbers)):  # 2番目から開始
        if numbers[i] > maximum:
            maximum = numbers[i]
    return maximum
  • バグ1: 空リストが渡されると numbers[0] で IndexError
  • 修正: if not numbers: で空チェックを追加
  • 改善: ループを range(1, ...) にして無駄な比較を削除

例題 3

効率性の改善（リスト vs セット）

Python
import time

def has_duplicate_slow(data):
    """重複チェック（リスト版 = 遅い O(n^2)）"""
    seen = []  # リストの検索は O(n)
    for item in data:
        if item in seen:  # O(n) の検索が n 回 → O(n^2)
            return True
        seen.append(item)
    return False

def has_duplicate_fast(data):
    """重複チェック（セット版 = 速い O(n)）"""
    seen = set()  # セットの検索は O(1)
    for item in data:
        if item in seen:  # O(1) の検索が n 回 → O(n)
            return True
        seen.add(item)
    return False

速度差の実例

データ数リスト版セット版速度差
1,0000.0123秒0.000089秒138倍
5,0000.3012秒0.000411秒733倍
10,0001.2048秒0.000823秒1,464倍

データ構造の選択だけで1000倍以上の速度差が生まれる！

例題 4

エラーハンドリングの追加

Python
def safe_divide(numerator, denominator):
    """安全に割り算をする関数"""
    try:
        result = numerator / denominator
        return result
    except ZeroDivisionError:
        print("  エラー: 0で割ることはできません")
        return None
    except TypeError:
        print("  エラー: 数値でない値が渡されました")
        return None

def validate_maze_input(maze, start, goal):
    """迷路の入力データを検証する関数"""
    errors = []
    if not maze:
        errors.append("迷路が空です")
        return errors
    rows = len(maze)
    cols = len(maze[0])
    sr, sc = start
    if not (0 <= sr < rows and 0 <= sc < cols):
        errors.append(f"スタート{start}が範囲外です")
    elif maze[sr][sc] != 0:
        errors.append(f"スタート{start}が壁です")
    return errors

バグのあるコードをトレースして問題を発見する

以下のコードを実行せずにトレースし、(1) 何が出力されるか、(2) どこにバグがあるかを答えなさい。

Python ── AL14-3.py
def remove_duplicates(data):
    seen = set()
    result = []
    for item in data:
        if item not in seen:
            result.append(item)
        seen.add(item)
    return result

def average(numbers):
    total = 0
    for n in numbers:
        total += n
    return total / len(numbers)

scores = [80, 90, 80, 70, 90, 60]
unique = remove_duplicates(scores)
print(f"元データ: {scores}")
print(f"重複除去: {unique}")
print(f"平均: {average(unique)}")
print(f"全体平均: {average(scores)}")

ヒント: remove_duplicates は正しく動作しますが、「重複を除去してから平均を取る」のは元の全体平均と同じでしょうか？ コードの論理的な問題を考えましょう。
