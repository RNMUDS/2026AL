説明

1.1 なぜソートが重要なのか

  • 検索の高速化: ソート済みデータなら二分探索 O(log n) が使える
  • データ分析: 中央値、最大値、最小値が簡単に見つかる
  • 重複の検出: ソート済みなら隣同士を比較するだけ
  • 表示の整理: 名前順、日付順、成績順など

1.2 挿入ソートの考え方

トランプのカードを手に持って、1枚ずつ正しい位置に差し込む動作と同じです。 最初のカードは「整列済み」（1枚だけなので）。 2枚目を取り、手札の中で正しい位置を見つけて差し込む。3枚目以降も同様。

1.3 アルゴリズムの手順

1.4 計算量

ケース比較回数特徴
最良（ソート済み）O(n)各要素で1回の比較のみ
平均O(n2)約 n(n-1)/4 回
最悪（逆順）O(n2)n(n-1)/2 回

data = [25, 34, 64, 12, 22]  # 先頭3つがソート済み
target_index = 3  # data[3]=12 を挿入する

key = data[target_index]
j = target_index - 1

while j >= 0 and data[j] > key:
    data[j + 1] = data[j]  # 要素を右にずらす
    j = j - 1

data[j + 1] = key  # keyを空いた位置に挿入
print(f"結果: {data}")  # → [12, 25, 34, 64, 22]
例題 4

完全な挿入ソート（比較回数カウント付き）

Python
def insertion_sort(data):
    n = len(data)
    comparison_count = 0
    shift_count = 0

    for i in range(1, n):
        key = data[i]
        j = i - 1

        while j >= 0:
            comparison_count += 1
            if data[j] > key:
                data[j + 1] = data[j]
                shift_count += 1
                j -= 1
            else:
                break

        data[j + 1] = key
        print(f"  ステップ{i}: key={key} → {data}")

    print(f"比較回数: {comparison_count}, 移動回数: {shift_count}")
    return data

test_data = [64, 34, 25, 12, 22]
insertion_sort(test_data[:])
例題 5

テキストベースのソート可視化

棒グラフ風のテキストで、挿入ソートの過程を可視化します。整列済み部分を #、挿入中を @、未整列を - で表示します。

Python
def visualize_sort(data):
    n = len(data)
    max_value = max(data)

    for i in range(1, n):
        key = data[i]
        j = i - 1
        while j >= 0 and data[j] > key:
            data[j + 1] = data[j]
            j -= 1
        data[j + 1] = key

        # 棒グラフ風に表示
        for k in range(n):
            bar_len = int(data[k] / max_value * 30)
            if k == j + 1:
                bar = "@" * bar_len
            elif k <= i:
                bar = "#" * bar_len
            else:
                bar = "-" * bar_len
            print(f"  data[{k:>2}]={data[k]:>3} |{bar}")
        print()

visualize_sort([64, 34, 25, 12, 22, 50, 15])

模範コード（AL08-4.py）:

# ===== 発展課題: 挿入ソートの安定性を確認する =====

def insertion_sort_by_value(data):
    """タプルの数値（2番目の要素）で挿入ソートする
    安定ソート: 同じ値の要素の元の順序を保つ"""
    result = data[:]  # コピーを作成（元データを変更しない）

    for i in range(1, len(result)):
        key = result[i]         # 挿入する要素
        j = i - 1

        # key の数値が result[j] の数値より小さい間、後ろにずらす
        # 「>」で比較 → 同じ値の場合は移動しない → 安定
        while j >= 0 and result[j][1] > key[1]:
            result[j + 1] = result[j]
            j -= 1

        result[j + 1] = key

    return result

# --- テストデータ ---
data = [("A", 3), ("B", 1), ("C", 3), ("D", 2)]

print("=== 挿入ソートの安定性確認 ===")
print(f"ソート前: {data}")

sorted_data = insertion_sort_by_value(data)
print(f"ソート後: {sorted_data}")

# --- 安定性の確認 ---
print()
print("--- 安定性のチェック ---")
# 同じ値(3)を持つ要素: A と C
# 元の順序: A が先、C が後
# 安定ソートなら: ソート後も A が C の前にある
same_value_elements = [(name, val) for name, val in sorted_data if val == 3]
print(f"値が3の要素の順序: {same_value_elements}")

if same_value_elements[0][0] == "A" and same_value_elements[1][0] == "C":
    print("→ 元の順序（A→C）が保たれている = 安定ソート!")
else:
    print("→ 元の順序が変わっている = 不安定ソート")

# --- 不安定ソートとの比較（選択ソート） ---
print()
print("--- 参考: 選択ソート（不安定）の場合 ---")

def selection_sort_by_value(data):
    """選択ソート（不安定）"""
    result = data[:]
    for i in range(len(result)):
        min_idx = i
        for j in range(i + 1, len(result)):
            if result[j][1] < result[min_idx][1]:
                min_idx = j
        result[i], result[min_idx] = result[min_idx], result[i]
    return result

sorted_selection = selection_sort_by_value(data)
print(f"選択ソート結果: {sorted_selection}")
same_sel = [(name, val) for name, val in sorted_selection if val == 3]
print(f"値が3の要素の順序: {same_sel}")

評価ポイント（これが説明できていればOK）:

  • 安定ソートとは「同じキー値を持つ要素の元の相対的な順序がソート後も保たれる」こと
  • 挿入ソートで >（厳密な大小）を使って比較すると、同じ値の場合は移動しないため安定になる
  • >= に変えると同じ値でも移動が起き、安定性が崩れることを理解している
  • 安定ソートが重要な場面（例: 名前でソート済みのデータを成績でソートし直す場合、同じ成績の人は名前順を保ちたい）が説明できる
