説明

1.1 なぜソートが重要なのか

  • 検索の高速化: ソート済みデータなら二分探索 O(log n) が使える
  • データ分析: 中央値、最大値、最小値が簡単に見つかる
  • 重複の検出: ソート済みなら隣同士を比較するだけ
  • 表示の整理: 名前順、日付順、成績順など

1.2 挿入ソートの考え方

トランプのカードを手に持って、1枚ずつ正しい位置に差し込む動作と同じです。 最初のカードは「整列済み」（1枚だけなので）。 2枚目を取り、手札の中で正しい位置を見つけて差し込む。3枚目以降も同様。

1.3 アルゴリズムの手順

1.4 計算量

ケース比較回数特徴
最良（ソート済み）O(n)各要素で1回の比較のみ
平均O(n2)約 n(n-1)/4 回
最悪（逆順）O(n2)n(n-1)/2 回

キーワード

この回の重要キーワード

用語説明
挿入ソート（Insertion Sort）未整列の要素を1つずつ取り出し、整列済み部分の正しい位置に「挿入」するソート。トランプの手札を並べる動作と同じ。
安定ソート（Stable Sort）同じ値を持つ要素の元の順序が保たれるソート。挿入ソートは安定ソートである。例: 同じ点数の学生の名前順が維持される。
計算量 O(n²)データ数 n の二乗に比例する時間がかかる。挿入ソートの平均・最悪計算量。n=1000 で約100万回の比較。
key（挿入する値）挿入ソートの各パスで「手に取った1枚のカード」に相当。この値を整列済み部分の正しい位置に差し込む。
図解

挿入ソート: カードを正しい位置に差し込む

例題 3

挿入ソートの1パス

Python
data = [25, 34, 64, 12, 22]  # 先頭3つがソート済み
target_index = 3  # data[3]=12 を挿入する

key = data[target_index]
j = target_index - 1

while j >= 0 and data[j] > key:
    data[j + 1] = data[j]  # 要素を右にずらす
    j = j - 1

data[j + 1] = key  # keyを空いた位置に挿入
print(f"結果: {data}")  # → [12, 25, 34, 64, 22]

例題 4

完全な挿入ソート（比較回数カウント付き）

Python
def insertion_sort(data):
    n = len(data)
    comparison_count = 0
    shift_count = 0

    for i in range(1, n):
        key = data[i]
        j = i - 1

        while j >= 0:
            comparison_count += 1
            if data[j] > key:
                data[j + 1] = data[j]
                shift_count += 1
                j -= 1
            else:
                break

        data[j + 1] = key
        print(f"  ステップ{i}: key={key} → {data}")

    print(f"比較回数: {comparison_count}, 移動回数: {shift_count}")
    return data

test_data = [64, 34, 25, 12, 22]
insertion_sort(test_data[:])

🖐 やってみよう（1分）

test_data を [1, 2, 3, 4, 5]（ソート済み）に変えて実行してみよう。比較回数はどうなる？

Notionに記録: ソート済みデータの比較回数と移動回数を記録してください。
例題 5

テキストベースのソート可視化

棒グラフ風のテキストで、挿入ソートの過程を可視化します。整列済み部分を #、挿入中を @、未整列を - で表示します。

Python
def visualize_sort(data):
    n = len(data)
    max_value = max(data)

    for i in range(1, n):
        key = data[i]
        j = i - 1
        while j >= 0 and data[j] > key:
            data[j + 1] = data[j]
            j -= 1
        data[j + 1] = key

        # 棒グラフ風に表示
        for k in range(n):
            bar_len = int(data[k] / max_value * 30)
            if k == j + 1:
                bar = "@" * bar_len
            elif k <= i:
                bar = "#" * bar_len
            else:
                bar = "-" * bar_len
            print(f"  data[{k:>2}]={data[k]:>3} |{bar}")
        print()

visualize_sort([64, 34, 25, 12, 22, 50, 15])

🖐 やってみよう（1分）

リストを [7, 6, 5, 4, 3, 2, 1]（逆順）に変えて実行してみよう。棒グラフの動きを観察して、最悪ケースの動きを確認しよう。

Notionに記録: 逆順データのとき、各パスでの移動の様子を観察して記録してください。

標準課題 2

挿入ソートコードのトレース

以下のコードを実行せずに、各パスで data がどう変化するかを追い、最終出力を予測しなさい。
予測を書いた後に、AL08-3.py として保存・実行して確かめなさい。

Python ── AL08-3.py
data = [5, 2, 8, 1, 4]
comparisons = 0

for i in range(1, len(data)):
    key = data[i]
    j = i - 1
    while j >= 0 and data[j] > key:
        data[j + 1] = data[j]
        j -= 1
        comparisons += 1
    data[j + 1] = key
    print(f"パス{i}: key={key} → {data}")

print(f"比較回数: {comparisons}")

ヒント: 各パスで key（挿入する値）がどこまで左に移動するかを追います。while 条件の「data[j] > key」が偽になる位置が挿入先です。
