説明

1.1 ハッシュテーブルとは

ハッシュテーブルは配列（スロット）とハッシュ関数の組み合わせです。

概念
キー "apple"  → ハッシュ関数 → インデックス 0 → テーブル[0] に格納
キー "banana" → ハッシュ関数 → インデックス 9 → テーブル[9] に格納
  • キーからハッシュ関数でインデックスを計算し、そのインデックスにデータを格納する
  • 検索・格納が平均 O(1) で行える高速なデータ構造

1.2 衝突（Collision）とは

異なるキーが同じインデックスに格納されようとすること。

  • テーブルサイズは有限（例: 10 スロット）
  • 格納するキーは無限に存在しうる
  • 鳩の巣原理: 11個のデータを10個のスロットに入れたら、必ずどこかが重複する

1.3 線形探索法（Linear Probing）

駐車場を想像してください。目的の番号の駐車場が埋まっていたら、隣の空いている場所に停めます。探すときも同じルートで探します。

衝突が起きたら次のスロットを試す。それも埋まっていたらさらに次へ。

1.4 チェイン法（Chaining）

各スロットにリストを持たせ、衝突したデータをリストに追加する方法。

概念
テーブル[3] → [("apple", 100), ("pleap", 999)]
テーブル[9] → [("banana", 80)]
比較線形探索法チェイン法
衝突時空きスロットを探すリストに追加
テーブル満杯時性能低下格納可能
メモリ配列のみリストの追加メモリ

1.5 Python の dict

Python の dict は内部的にハッシュテーブルを使っています。hash() 組み込み関数でハッシュ値を確認できます。

Python
print(hash("apple"))   # Pythonが計算するハッシュ値
print(hash("banana"))  # 毎回同じ値（決定性）

table_size = 10
table = [None] * table_size  # 全スロットをNoneで初期化

    total = 0
    for char in key:
        total = total + ord(char)
    return total % size

例題 4

線形探索法で格納と検索

Python
def insert_with_probing(table, key, value):
    size = len(table)
    index = hash_function(key, size)
    original_index = index
    probe_count = 0
    while table[index] is not None:
        if table[index][0] == key:
            break
        probe_count += 1
        index = (index + 1) % size
        if index == original_index:
            print("テーブルが満杯です")
            return -1
    table[index] = (key, value)
    print(f"  insert '{key}': 初期={original_index}, 探索{probe_count}回 → [{index}]")

def search_with_probing(table, key):
    size = len(table)
    index = hash_function(key, size)
    start_index = index
    probe_count = 0
    while table[index] is not None:
        probe_count += 1
        if table[index][0] == key:
            print(f"  search '{key}': [{index}]で発見, 探索{probe_count}回")
            return table[index][1]
        index = (index + 1) % size
        if index == start_index:
            break
    print(f"  search '{key}': 見つかりません")
    return None

table = [None] * 10
insert_with_probing(table, "apple", 100)
insert_with_probing(table, "pleap", 999)  # appleと衝突
search_with_probing(table, "apple")
search_with_probing(table, "pleap")
例題 5

HashTable クラスの作成

insert / search / delete / display メソッドを持つクラスを作成します。

Python
class HashTable:
    def __init__(self, size=10):
        self.size = size
        self.table = [None] * size
        self.count = 0

    def _hash(self, key):
        total = 0
        for char in key:
            total += ord(char)
        return total % self.size

    def insert(self, key, value):
        index = self._hash(key)
        original = index
        while self.table[index] is not None:
            if self.table[index][0] == key:
                break
            index = (index + 1) % self.size
            if index == original:
                return -1
        self.table[index] = (key, value)
        self.count += 1

    def search(self, key):
        index = self._hash(key)
        start = index
        while self.table[index] is not None:
            if self.table[index][0] == key:
                return self.table[index][1]
            index = (index + 1) % self.size
            if index == start: break
        return None

    def delete(self, key):
        index = self._hash(key)
        start = index
        while self.table[index] is not None:
            if self.table[index][0] == key:
                self.table[index] = None
                self.count -= 1
                return True
            index = (index + 1) % self.size
            if index == start: break
        return False

# --- テスト ---
ht = HashTable(10)
ht.insert("apple", 100)
ht.insert("banana", 80)
print(ht.search("apple"))   # → 100
ht.delete("apple")
print(ht.search("apple"))   # → None

# ===== 発展課題: ハッシュテーブルの負荷率と性能 =====

import random

class HashTable:
    """線形探索法（オープンアドレス法）のハッシュテーブル"""

    def __init__(self, size):
        self.size = size
        self.table = [None] * size

    def _hash(self, key):
        """多項式ハッシュ関数"""
        h = 0
        for c in str(key):
            h = h * 31 + ord(c)
        return h % self.size

    def insert(self, key):
        """データを挿入"""
        index = self._hash(key)
        for i in range(self.size):
            pos = (index + i) % self.size
            if self.table[pos] is None:
                self.table[pos] = key
                return
        print("テーブルが満杯です")

    def search(self, key):
        """データを検索し、比較回数を返す"""
        index = self._hash(key)
        for i in range(self.size):
            pos = (index + i) % self.size
            if self.table[pos] is None:
                return i + 1  # 見つからない
            if self.table[pos] == key:
                return i + 1  # 見つかった（比較回数）
        return self.size  # テーブル全体を探索

def experiment(load_factor, table_size=1000):
    """指定された負荷率で実験"""
    ht = HashTable(table_size)
    n = int(table_size * load_factor)  # 格納するデータ数

    # ランダムなデータを格納
    keys = [f"key_{random.randint(0, 100000)}" for _ in range(n)]
    for key in keys:
        ht.insert(key)

    # 格納したデータを全て検索して平均比較回数を計算
    total_count = 0
    for key in keys:
        total_count += ht.search(key)

    avg_count = total_count / len(keys)
    return avg_count, n

# --- 実験実行 ---
print("=== ハッシュテーブルの負荷率と平均検索回数 ===")
print(f"{'負荷率':>8} | {'データ数':>8} | {'平均検索回数':>12}")
print("-" * 38)

for lf in [0.5, 0.75, 0.9]:
    avg, n = experiment(lf)
    print(f"{lf:>8.2f} | {n:>8} | {avg:>12.2f}")

print()
print("理論値（線形探索法）:")
print("  負荷率 a のとき、平均検索回数 ≒ 1/(1-a)")
for lf in [0.5, 0.75, 0.9]:
    theory = 1 / (1 - lf)
    print(f"  a={lf}: 約 {theory:.2f} 回")

評価ポイント（これが説明できていればOK）:

  • 負荷率（Load Factor）= データ数 / テーブルサイズ であり、テーブルの埋まり具合を表す
  • 負荷率が高くなると衝突が増え、線形探索法ではクラスタリング（連続した塊）が発生して検索回数が増加する
  • 負荷率 0.5 では平均約2回、0.75 では約4回、0.9 では約10回と急激に増加する
  • 実用では負荷率 0.7〜0.75 程度を上限とし、それを超えたらテーブルを拡張（リハッシュ）するのが一般的
