説明: 探索とは何か

「探索（Search）」とは、データの集まりの中から、目的のデータを見つける操作です。

日常生活の「探索」

場面やっていること
本棚から本を探す1冊ずつ背表紙を見て、目的の本か確認する
電話帳で名前を探すページをめくりながら名前を探す
出席簿で名前を探す上から順に名前を確認する
LINEで友達を探す友達リストを上からスクロールする

逐次探索（線形探索）とは

出席簿で特定の名前を探すとき、1行目から順に読んでいく ── これが逐次探索です。最初に見つかったらそこで終わり、最後まで見つからなければ「いない」と分かります。

やり方: データの先頭から順に、1つずつ目的の値と比較していく。

計算量 O(n) とは

O(n) の読み方・表し方

表記読み方意味
O(n)「オーダー n」 または 「ビッグオー n」データ数 n に比例する計算量
O(1)「オーダー 1」データ数に関係なく一定（例: 辞書のキー検索）
O(n2)「オーダー n の2乗」データ数の2乗に比例（例: バブルソート）
O(log n)「オーダー ログ n」データ数の対数に比例（例: 二分探索）

「O」は英語の Order（オーダー） の頭文字で、ビッグオー記法（Big-O notation）と呼ばれます。アルゴリズムの効率を「最悪の場合、どれくらい時間がかかるか」で表す記法です。定数や小さい項は省略し、データ量が大きくなったときに支配的な部分だけを残します。

例: 比較回数が 3n + 5 回なら → O(n)。n が十分大きいとき +5 や 3× は誤差になるため省略します。

100ページの本を1ページずつめくって探すと最悪100回。1000ページなら最悪1000回。ページ数が増えた分だけ、探す時間も増えます。
データ数 (n)最悪比較回数平均比較回数
1010回5回
100100回50回
1,0001,000回500回
1,000,0001,000,000回500,000回

データ量が2倍になると、探索時間も2倍になる ── これが O(n) の意味です。

例題 1

リストの中から値を探す

逐次探索の最も基本的な形です。

Python
data = [3, 7, 1, 9, 4]   # 探す対象のデータ（リスト）
target = 9                       # 探したい値

print(f"データ: {data}")          # データの中身を表示
print(f"探す値: {target}")        # 探す値を表示

found = False                    # 見つかったかどうかのフラグ（最初は「まだ見つかっていない」）

for i in range(len(data)):      # i = 0, 1, 2, ... と先頭から順番に調べる
    print(f"  ステップ{i + 1}: data[{i}] = {data[i]}", end="")  # 今見ている要素を表示

    if data[i] == target:        # 今見ている要素が探す値と同じ？
        print(f" → {target}と一致！ 発見！")
        found = True             # 見つかった！フラグを True に
        found_index = i          # 見つかった位置（インデックス）を記録
        break                    # 見つかったのでループを抜ける
    else:                        # 一致しなかった場合
        print(f" → {target}と不一致 → 次へ")

if found:                        # ループ終了後、見つかっていれば結果を表示
    print(f"結果: インデックス {found_index} で発見")
else:                            # 最後まで見つからなかった場合
    print(f"結果: {target} は見つかりませんでした")

処理の流れ: data = [3, 7, 1, 9, 4] から target = 9 を探す

例題 2

関数にまとめた逐次探索

逐次探索を関数として定義し、インデックスと比較回数を返すようにします。

Python
def linear_search(data, target):   # 関数の定義（データと探す値を受け取る）
    """逐次探索: (インデックス, 比較回数) を返す"""
    comparison_count = 0            # 比較した回数を数えるカウンター

    for i in range(len(data)):      # 先頭から順番に1つずつ調べる
        comparison_count = comparison_count + 1  # 比較するたびにカウンター +1

        if data[i] == target:        # 今見ている要素が探す値と一致したら
            return (i, comparison_count)  # 見つかった位置と比較回数を返して終了

    return (-1, comparison_count)  # 最後まで見つからなかったら -1 を返す

# --- テスト ---
test_data = [15, 3, 27, 8, 42, 11, 36, 19, 5, 22]  # 10個のデータ

for target in [42, 15, 22, 100]:  # 4つの値を順番に探す
    index, count = linear_search(test_data, target)  # 関数を呼び出して探索
    if index != -1:              # 見つかった場合（-1 でない）
        print(f"  target={target:>3} → インデックス{index}で発見 (比較{count}回)")
    else:                        # 見つからなかった場合
        print(f"  target={target:>3} → 見つからず (比較{count}回)")

出力例: target=42 は比較5回で発見、target=100 は比較10回で未発見

やってみよう（1分）

test_data の先頭に 42 を追加して [42, 15, 3, 27, ...] にしてから、target=42 を探してみよう。

→ 比較回数はどう変わった？ 位置が変わると回数も変わることを確認しよう。

Notionに記録: 変更前後の target=42 の比較回数を記録してください。
例題 3

文字列の中から文字を探す

逐次探索は文字列の中の文字探索にも使えます。

Python
def search_char_in_string(text, target_char):  # 文字列と探す文字を受け取る関数
    """文字列の中から特定の文字を探す"""
    positions = []                # 見つかった位置を入れるリスト（最初は空）
    comparison_count = 0          # 比較回数カウンター

    for i in range(len(text)):    # 文字列を先頭から1文字ずつ調べる
        comparison_count = comparison_count + 1  # 比較するたびに +1
        if text[i] == target_char:   # 今の文字が探す文字と一致？
            positions.append(i)    # 一致！→ 位置をリストに追加
            print(f"  位置{i}: '{text[i]}' → 一致！")
        else:                        # 一致しなかった場合
            print(f"  位置{i}: '{text[i]}' → 不一致")

    return (positions, comparison_count)  # 全位置と比較回数を返す（breakしない＝全部調べる）

# --- テスト ---
text = "banana"                  # 探す対象の文字列
target_char = "a"                # 探す文字
positions, count = search_char_in_string(text, target_char)  # 関数を呼び出し
print(f"結果: '{target_char}' は位置 {positions} に見つかりました")
# 結果: 'a' は位置 [1, 3, 5] に見つかりました

やってみよう（1分）

target_char = "a" を target_char = "n" に変えて実行してみよう。

→ "n" は何箇所に見つかる？ positions リストの中身を予測してから実行しよう。

Notionに記録: target_char="n" のときの positions と comparison_count を記録してください。
例題 4

比較回数を記録する逐次探索

比較回数を詳細に記録して、アルゴリズムの効率を分析します。

Python
# ※ この例題は例題2の linear_search 関数を使います（先に例題2を実行しておくこと）
data = [15, 3, 27, 8, 42, 11, 36, 19, 5, 22]  # 10個のデータ
total_comparisons = 0            # 全探索の比較回数の合計

for target in data:              # データの各要素を順番に探す（全員を1回ずつ探す）
    index, count = linear_search(data, target)  # 例題2の関数で探索
    total_comparisons = total_comparisons + count  # 合計に加算
    print(f"  target={target:>2} → インデックス{index}, 比較{count:>2}回")

n = len(data)                    # データの個数（= 10）
average = total_comparisons / n  # 合計 ÷ 個数 = 平均比較回数
print(f"平均比較回数: {average}")       # 実測値: 5.5
print(f"理論上の平均 (n+1)/2: {(n+1)/2}")  # 理論値: (10+1)/2 = 5.5 → 一致！

やってみよう（1分）

print(f"最悪比較回数: {n}") を最後に追加して、最悪ケースの回数も表示してみよう。

→ 平均と最悪の差はどれくらい？ n が大きくなるとこの差はどうなる？

Notionに記録: 平均比較回数と最悪比較回数の値、そして (n+1)/2 の式が正しいか確認した結果を記録してください。
例題 5

大規模データでの探索時間測定

データサイズを変えて探索時間を測定し、O(n) の性質を体感します。

Python
import random                     # ランダムな数を作るための道具
import time                       # 時間を計るための道具

def linear_search_simple(data, target):  # シンプルな逐次探索（結果だけ返す）
    for i in range(len(data)):  # 先頭から順番に調べる
        if data[i] == target:    # 見つかったら
            return i             # その位置を返す
    return -1                   # 最後まで見つからなかったら -1

sizes = [1000, 10000, 100000]  # 試すデータサイズ（1千, 1万, 10万）
previous_time = None             # 前回の実行時間（倍率計算用）

for size in sizes:               # 各サイズについて実験する
    data = [random.randint(0, size * 10) for _ in range(size)]  # ランダムなデータを生成
    target = -1                  # 絶対に存在しない値 → 最悪ケース（全部調べる）

    start_time = time.time()     # 計測開始
    for _ in range(100):        # 100回繰り返して合計時間を計る（精度のため）
        linear_search_simple(data, target)
    elapsed = time.time() - start_time  # 計測終了 → かかった秒数

    if previous_time and previous_time > 0:  # 2回目以降は倍率も計算
        ratio = elapsed / previous_time  # 今回 ÷ 前回 = 何倍遅くなったか
        print(f"  {size:>8,}件  {elapsed:.3f}秒  {ratio:.1f}倍")
    else:                        # 1回目は倍率なし
        print(f"  {size:>8,}件  {elapsed:.3f}秒")
    previous_time = elapsed      # 今回の時間を記録（次の倍率計算に使う）

# 考察: データが10倍 → 時間もおよそ10倍 → O(n) の証拠！

やってみよう（1分）

sizes に 1000000（100万）を追加して実行してみよう。倍率は約10倍になるか？

→ 結果の「倍」の値が約10.0に近いかどうか確認しよう。これが O(n) の証拠。

Notionに記録: 各データサイズの実行時間と倍率を記録してください。

以下のコードを実行せずに、ループの各ステップと最終出力を予測しなさい。

Python ── AL03-3.py
data = [15, 3, 27, 8, 42, 11, 36, 19, 5, 22]
targets = [8, 100, 15]
total = 0

for t in targets:
    count = 0
    found = False
    for d in data:
        count += 1
        if d == t:
            found = True
            break
    total += count
    print(f"target={t}: {count}回, found={found}")

print(f"合計比較回数: {total}")

ヒント: 各 target について、data のどの位置で見つかるか（または見つからないか）を考え、そのときの count を数えましょう。

解答例

例題 例題1 やってみよう 解答

target = 99 のとき（data = [3, 7, 1, 9, 4]）:

ステップ1: data[0]=3, 3!=99
ステップ2: data[1]=7, 7!=99
ステップ3: data[2]=1, 1!=99
ステップ4: data[3]=9, 9!=99
ステップ5: data[4]=4, 4!=99
結果: found = False, ステップ数 = 5

99はリストに存在しないため、全要素（5個）を比較して found=False。

例題 例題2 やってみよう 解答

先頭に42を追加: [42, 15, 3, 27, 8, 42, 11, 36, 19, 5, 22]

変更前: target=42 は元のインデックス4にあるので比較回数 = 5回

変更後: target=42 は先頭（インデックス0）にあるので比較回数 = 1回

逐次探索は先頭から順に調べるため、要素の位置が前にあるほど速く見つかる。

例題 例題3 やってみよう 解答

target_char = "n", text = "banana" のとき:

positions = [2, 4]  （"banana" のインデックス2と4に "n" がある）
comparison_count = 6  （全文字を比較）

"n" は2箇所に見つかる。全文字探索なので比較回数は文字数と同じ6回。

例題 例題4 やってみよう 解答

n個のデータでの逐次探索の比較回数:

平均比較回数 ≒ (n+1)/2
最悪比較回数 = n

例えば n=100 なら、平均 ≒ 50.5回、最悪 = 100回。(n+1)/2 の式が正しいことが確認できる。

例題 例題5 やってみよう 解答

sizes に 1000000 を追加した場合の結果例:

データサイズ実行時間倍率（前のサイズとの比）
1,000約0.001秒—
10,000約0.01秒約10倍
100,000約0.1秒約10倍
1,000,000約1.0秒約10倍

データが10倍になると実行時間も約10倍 → O(n) の証拠。
