説明: 探索とは何か

「探索（Search）」とは、データの集まりの中から、目的のデータを見つける操作です。

日常生活の「探索」

場面やっていること
本棚から本を探す1冊ずつ背表紙を見て、目的の本か確認する
電話帳で名前を探すページをめくりながら名前を探す
出席簿で名前を探す上から順に名前を確認する
LINEで友達を探す友達リストを上からスクロールする

逐次探索（線形探索）とは

出席簿で特定の名前を探すとき、1行目から順に読んでいく ── これが逐次探索です。最初に見つかったらそこで終わり、最後まで見つからなければ「いない」と分かります。

やり方: データの先頭から順に、1つずつ目的の値と比較していく。

計算量 O(n) とは

100ページの本を1ページずつめくって探すと最悪100回。1000ページなら最悪1000回。ページ数が増えた分だけ、探す時間も増えます。
データ数 (n)最悪比較回数平均比較回数
1010回5回
100100回50回
1,0001,000回500回
1,000,0001,000,000回500,000回

データ量が2倍になると、探索時間も2倍になる ── これが O(n) の意味です。

例題 1

リストの中から値を探す

逐次探索の最も基本的な形です。

Python
data = [3, 7, 1, 9, 4]
target = 9

print(f"データ: {data}")
print(f"探す値: {target}")

found = False

for i in range(len(data)):
    print(f"  ステップ{i + 1}: data[{i}] = {data[i]}", end="")

    if data[i] == target:
        print(f" → {target}と一致！ 発見！")
        found = True
        found_index = i
        break
    else:
        print(f" → {target}と不一致 → 次へ")

if found:
    print(f"結果: インデックス {found_index} で発見")
else:
    print(f"結果: {target} は見つかりませんでした")
例題 2

関数にまとめた逐次探索

逐次探索を関数として定義し、インデックスと比較回数を返すようにします。

Python
def linear_search(data, target):
    """逐次探索: (インデックス, 比較回数) を返す"""
    comparison_count = 0

    for i in range(len(data)):
        comparison_count = comparison_count + 1

        if data[i] == target:
            return (i, comparison_count)

    return (-1, comparison_count)

# --- テスト ---
test_data = [15, 3, 27, 8, 42, 11, 36, 19, 5, 22]

for target in [42, 15, 22, 100]:
    index, count = linear_search(test_data, target)
    if index != -1:
        print(f"  target={target:>3} → インデックス{index}で発見 (比較{count}回)")
    else:
        print(f"  target={target:>3} → 見つからず (比較{count}回)")

出力例: target=42 は比較5回で発見、target=100 は比較10回で未発見

例題 3

文字列の中から文字を探す

逐次探索は文字列の中の文字探索にも使えます。

Python
def search_char_in_string(text, target_char):
    """文字列の中から特定の文字を探す"""
    positions = []
    comparison_count = 0

    for i in range(len(text)):
        comparison_count = comparison_count + 1
        if text[i] == target_char:
            positions.append(i)
            print(f"  位置{i}: '{text[i]}' → 一致！")
        else:
            print(f"  位置{i}: '{text[i]}' → 不一致")

    return (positions, comparison_count)

# --- テスト ---
text = "banana"
target_char = "a"
positions, count = search_char_in_string(text, target_char)
print(f"結果: '{target_char}' は位置 {positions} に見つかりました")
# 結果: 'a' は位置 [1, 3, 5] に見つかりました
例題 4

比較回数を記録する逐次探索

比較回数を詳細に記録して、アルゴリズムの効率を分析します。

Python
data = [15, 3, 27, 8, 42, 11, 36, 19, 5, 22]
total_comparisons = 0

for target in data:
    index, count = linear_search(data, target)
    total_comparisons = total_comparisons + count
    print(f"  target={target:>2} → インデックス{index}, 比較{count:>2}回")

n = len(data)
average = total_comparisons / n

print(f"理論上の平均 (n+1)/2: {(n+1)/2}")  # 5.5
例題 5

大規模データでの探索時間測定

データサイズを変えて探索時間を測定し、O(n) の性質を体感します。

Python
import random
import time

def linear_search_simple(data, target):
    for i in range(len(data)):
        if data[i] == target:
            return i
    return -1

sizes = [1000, 10000, 100000]
previous_time = None

for size in sizes:
    data = [random.randint(0, size * 10) for _ in range(size)]
    target = -1  # 存在しない値で最悪ケース

    start_time = time.time()
    for _ in range(100):
        linear_search_simple(data, target)
    elapsed = time.time() - start_time

    if previous_time and previous_time > 0:
        ratio = elapsed / previous_time
        print(f"  {size:>8,}件  {elapsed:.3f}秒  {ratio:.1f}倍")
    else:
        print(f"  {size:>8,}件  {elapsed:.3f}秒")
    previous_time = elapsed

# 考察: データが10倍 → 時間もおよそ10倍 → O(n)

模範コード（AL03-4.py）:

# ===== 発展課題: 逐次探索の最適化（Move-to-Front法） =====

import random

# --- 通常の逐次探索 ---
def linear_search(data, target):
    """通常の逐次探索: 比較回数を返す"""
    for i in range(len(data)):
        if data[i] == target:
            return i + 1  # 比較回数
    return len(data)  # 見つからない場合

# --- Move-to-Front法の逐次探索 ---
def mtf_search(data, target):
    """Move-to-Front法: 見つけた要素を先頭に移動し、比較回数を返す"""
    for i in range(len(data)):
        if data[i] == target:
            # 見つけた要素を先頭に移動
            found = data.pop(i)  # 取り出す
            data.insert(0, found)  # 先頭に挿入
            return i + 1  # 比較回数
    return len(data)

# --- テストデータの準備 ---
n = 100  # リストのサイズ
original = list(range(n))  # 0〜99のリスト

# よく検索される要素（偏りのある検索パターン）
# 0〜9を多めに検索する
search_targets = []
for _ in range(500):
    if random.random() < 0.7:  # 70%の確率で0〜9を検索
        search_targets.append(random.randint(0, 9))
    else:
        search_targets.append(random.randint(0, n - 1))

# --- 通常の逐次探索で実験 ---
data_normal = original[:]  # コピーを作成
total_normal = 0
for target in search_targets:
    total_normal += linear_search(data_normal, target)

# --- Move-to-Front法で実験 ---
data_mtf = original[:]  # コピーを作成
total_mtf = 0
for target in search_targets:
    total_mtf += mtf_search(data_mtf, target)

# --- 結果の表示 ---
print(f"検索回数: {len(search_targets)}回")
print(f"通常の逐次探索: 合計比較回数 = {total_normal}")
print(f"Move-to-Front法: 合計比較回数 = {total_mtf}")
print(f"比較回数の削減率: {(1 - total_mtf / total_normal) * 100:.1f}%")

評価ポイント（これが説明できていればOK）:

  • Move-to-Front法は「見つけた要素を先頭に移動する」ことで、頻繁に検索される要素が前方に集まる仕組み
  • 検索パターンに偏りがある場合、通常の逐次探索より比較回数が大幅に減少する
  • 検索パターンが均一（偏りがない）場合は効果が薄い、または逆効果になることもある
  • 要素の移動コスト（pop と insert）がかかるため、リストが大きいと移動自体に時間がかかる点がデメリット
