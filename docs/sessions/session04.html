説明: 二分探索とは

辞書引きのたとえ

国語辞典で「もも」を調べるとき、1ページ目から順に探しますか? いいえ。真ん中あたりを開いて「さ行」だった → 「も」はもっと後ろ → 後半を開く…。この「半分に絞り込む」作業を繰り返すのが二分探索です。

二分探索の条件

重要: 二分探索はソート済み（整列済み）のデータにしか使えません。

例
使える:   [1, 3, 5, 7, 9, 11, 13]  ← ソート済み
使えない: [3, 7, 1, 9, 4]          ← バラバラ

アルゴリズムの手順

  • left = 0（先頭）、right = n-1（末尾）に設定
  • mid = (left + right) // 2 で中央を計算
  • data[mid] と target を比較:
      一致 → 発見!   小さい → 右半分へ   大きい → 左半分へ
  • left > right になったら「見つからない」

計算量 O(log n) とは

二分探索は毎回データを半分に絞り込みます。

データ数 (n)最大比較回数 (log2 n)
104回
1007回
1,00010回
100,00017回
1,000,00020回

100万件のデータから、たった20回の比較で見つけられる ── 逐次探索の50,000倍速い!

例題 1

ソート済みリストの基本

Python
numbers = [5, 2, 8, 1, 9, 3, 7, 4, 6]
print(f"元のリスト: {numbers}")

sorted_numbers = sorted(numbers)
print(f"ソート後:   {sorted_numbers}")

# ソート済みか判定する関数
def is_sorted(data):
    for i in range(len(data) - 1):
        if data[i] > data[i + 1]:
            return False
    return True

print(f"元はソート済み？: {is_sorted(numbers)}")       # False
print(f"ソート後はソート済み？: {is_sorted(sorted_numbers)}")  # True
例題 2

二分探索のステップ表示

各ステップで left, right, mid の変化を追跡します。

Python
def binary_search_trace(data, target):
    """二分探索を実行し、各ステップの状態を表示する"""
    left = 0
    right = len(data) - 1
    step = 0

    print(f"探索対象: {data}")
    print(f"target: {target}")

    while left <= right:
        step += 1
        mid = (left + right) // 2

        print(f"ステップ{step}: left={left}, right={right}, mid={mid}, data[{mid}]={data[mid]}")

        if data[mid] == target:
            print(f"  → 発見! インデックス{mid}")
            return mid
        elif data[mid] < target:
            print(f"  → {data[mid]} < {target} なので右半分へ")
            left = mid + 1
        else:
            print(f"  → {data[mid]} > {target} なので左半分へ")
            right = mid - 1

    print(f"  → 見つかりませんでした（{step}回比較）")
    return -1

# --- テスト ---
data = [1, 3, 5, 7, 9, 11, 13]
binary_search_trace(data, 11)
binary_search_trace(data, 6)  # 存在しない値
例題 3

逐次探索と二分探索の比較回数

同じデータで逐次探索と二分探索の比較回数を比べます。

Python
def linear_search_count(data, target):
    count = 0
    for i in range(len(data)):
        count += 1
        if data[i] == target:
            return (i, count)
    return (-1, count)

def binary_search_count(data, target):
    left, right, count = 0, len(data) - 1, 0
    while left <= right:
        count += 1
        mid = (left + right) // 2
        if data[mid] == target:
            return (mid, count)
        elif data[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return (-1, count)

# --- 20個のソート済みデータで比較 ---
data = list(range(1, 21))
print(f"{'target':>6}  {'逐次':>8}  {'二分':>8}  {'差':>6}")
for target in [1, 5, 10, 15, 20, 25]:
    _, cnt_l = linear_search_count(data, target)
    _, cnt_b = binary_search_count(data, target)
    print(f"{target:>6}  {cnt_l:>5}回  {cnt_b:>5}回  {cnt_l-cnt_b:>+5}回")
例題 4

bisect モジュールの活用

Python標準ライブラリの bisect モジュールで二分探索を使います。

Python
import bisect

data = [2, 5, 8, 12, 16, 23, 38, 42, 56, 72, 91, 100]

# --- 存在する値を探す ---
target = 23
pos = bisect.bisect_left(data, target)
if pos < len(data) and data[pos] == target:
    print(f"data[{pos}] = {target} → 存在する!")

# --- 重複ありのデータ ---
data_dup = [1, 3, 5, 5, 5, 7, 9]
left_pos = bisect.bisect_left(data_dup, 5)   # 最初の5の位置 → 2
right_pos = bisect.bisect_right(data_dup, 5)  # 最後の5の次 → 5
print(f"5の個数: {right_pos - left_pos}個")    # 3個
例題 5

ベンチマーク（性能比較）

10万件のデータで逐次探索と二分探索の実行時間を比較します。

Python
import time, random

n = 100000
data = list(range(n))
targets = [random.randint(0, n-1) for _ in range(500)]

# 二分探索の計測
start = time.perf_counter()
for t in targets:
    binary_search(data, t)
time_binary = time.perf_counter() - start

# 逐次探索の計測
start = time.perf_counter()
for t in targets:
    linear_search(data, t)
time_linear = time.perf_counter() - start

print(f"二分探索: {time_binary:.4f}秒")
print(f"逐次探索: {time_linear:.4f}秒")
print(f"二分探索は約 {time_linear/time_binary:,.0f}倍 速い!")
