説明

1.1 ハッシュとは何か？

大学のロッカーを想像してください。ロッカーには番号がついている（0, 1, 2, ..., 99）。 名前からロッカー番号を計算するルールがある。 番号がわかれば、一発でそのロッカーを開けられる（O(1)）。

これがハッシュの基本です。キーからインデックスを計算し、直接アクセスします。

操作線形探索ハッシュ
データを探す先頭から1つずつ調べる番号を計算して直接アクセス
計算量O(n)O(1)（理想的な場合）
例え本棚を端から探すロッカー番号で直接開ける

1.2 ハッシュ関数の3つの性質

  • 決定性: 同じ入力 → 必ず同じ出力
  • 一様分布: 出力がまんべんなく散らばる
  • 高速: 計算が速い

1.3 単純ハッシュと多項式ハッシュ

単純ハッシュ: 各文字の ord 値を合計し、テーブルサイズで割った余り

概念
hash("apple") = (ord('a') + ord('p') + ord('p') + ord('l') + ord('e')) % テーブルサイズ

問題点: 文字の順番を考慮しないので、アナグラム（"apple" と "pleap"）が同じ値になる。

多項式ハッシュ: 基数の累乗をかけて位置情報を反映させる

概念
h = 0
h = h * 31 + ord(c0)
h = h * 31 + ord(c1)
...

利点: 文字の順番も考慮するので、アナグラムでも異なる値になる。

用語説明
ハッシュ関数キー（文字列や数値）を受け取り、テーブルのインデックス（0〜サイズ-1）を返す関数。同じキーからは必ず同じインデックスが得られる（決定性）。
衝突（Collision）異なるキーが同じインデックスに割り当てられること。テーブルサイズが有限なので、十分な数のキーを入れると必ず発生する（鳩の巣原理）。
テーブルサイズハッシュテーブルのスロット数。素数にすると衝突が減りやすい。hash(key) % テーブルサイズ でインデックスを決める。
ord() / chr()ord('A') は文字をUnicodeコードポイント（数値）に変換。chr(65) は逆変換。ハッシュ値の計算に使う。
例題 1

ord() 関数で文字を数値にする

Pythonの ord() 関数で文字をUnicodeの数値に変換し、chr() で逆変換できます。

Python
# --- アルファベットの ord 値 ---
print(f"'A' → {ord('A')}")  # 65
print(f"'a' → {ord('a')}")  # 97
print(f"'あ' → {ord('あ')}")  # 12354

# --- 逆変換: chr() ---
print(f"chr(65) = '{chr(65)}'")  # A

# --- 文字列の各文字を数値に変換する ---
name = "Python"
total = 0
for i, char in enumerate(name):
    code = ord(char)
    total += code
    print(f"  位置{i}: '{char}' → {code}  (累計: {total})")
print(f"合計: {total}")

例題 2

単純ハッシュ関数

各文字の ord 値を合計し、テーブルサイズで割った余りを返します。

Python
def simple_hash(key, table_size):
    total = 0
    for char in key:
        code = ord(char)
        total += code
    hash_value = total % table_size
    return hash_value

# --- テスト ---
table_size = 10
test_words = ["apple", "banana", "cherry", "date", "elderberry"]
for word in test_words:
    h = simple_hash(word, table_size)
    print(f"  simple_hash('{word}', {table_size}) = {h}")

# --- アナグラムの問題点 ---
h1 = simple_hash("apple", table_size)
h2 = simple_hash("pleap", table_size)
print(f"apple={h1}, pleap={h2} → 同じ値になる!")

例題 3

多項式ハッシュ関数

基数（base=31）の累乗をかけて、文字の位置情報を反映させます。

Python
def polynomial_hash_trace(key):
    base = 31
    mask = 0xFFFFFFFF  # 32ビットマスク
    h = 0
    print(f"'{key}' の多項式ハッシュ計算（base={base}）")

    for step, char in enumerate(key, 1):
        prev = h
        code = ord(char)
        calculated = prev * base + code
        h = calculated & mask  # 32ビットで折りたたむ
        print(f"  Step{step}: '{char}' ord={code}  {prev}*{base}+{code} = {h}")

    print(f"最終ハッシュ値: {h} (0x{h:08X})")
    return h

polynomial_hash_trace("apple")
print()
polynomial_hash_trace("pleap")
# → 多項式ハッシュでは異なる値になる!

例題 4

複数の名前でハッシュ値を比較する

日本の名字トップ15で単純ハッシュと多項式ハッシュの分布を比較します。

Python
def simple_hash(key, table_size):
    total = 0
    for char in key:
        total += ord(char)
    return total % table_size

def polynomial_hash(key, base=31, mask=0xFFFFFFFF):
    h = 0
    for char in key:
        h = (h * base + ord(char)) & mask
    return h

names = ["Sato", "Suzuki", "Takahashi", "Tanaka", "Watanabe",
         "Ito", "Yamamoto", "Nakamura", "Kobayashi", "Kato"]
table_size = 10

for name in names:
    s = simple_hash(name, table_size)
    p = polynomial_hash(name) % table_size
    print(f"{name:<14} 単純={s}  多項式={p}")

例題 5

衝突の検出

多数の単語をハッシュテーブルに入れたとき、同じスロットに複数の単語が割り当てられる（衝突する）ことを確認します。

Python
def simple_hash(key, table_size):
    total = 0
    for char in key:
        total += ord(char)
    return total % table_size

words = ["apple", "banana", "cherry", "date", "elderberry",
         "fig", "grape", "honeydew", "kiwi", "lemon",
         "mango", "nectarine", "orange", "papaya", "quince",
         "raspberry", "strawberry", "tangerine", "ugli", "vanilla"]
table_size = 10

slots = {}
for word in words:
    h = simple_hash(word, table_size)
    if h not in slots:
        slots[h] = []
    slots[h].append(word)

for slot_num in sorted(slots.keys()):
    words_in_slot = slots[slot_num]
    marker = " ← 衝突!" if len(words_in_slot) > 1 else ""
    print(f"  スロット{slot_num}: {words_in_slot}{marker}")

標準課題 2

ハッシュテーブル挿入コードのトレース

以下のコードを実行せずに、ループの各ステップで table がどう変化するかを追い、最終出力を予測しなさい。
予測を書いた後に、AL06-3.py として保存・実行して確かめなさい。

Python ── AL06-3.py
SIZE = 7
table = [None] * SIZE
names = ["Alice", "Bob", "Nancy", "Tom"]
collisions = 0

for name in names:
    h = ord(name[0]) % SIZE
    if table[h] is None:
        table[h] = name
        print(f"{name}: hash={h} → 格納")
    else:
        collisions += 1
        print(f"{name}: hash={h} → 衝突！({table[h]}が既にある)")

print(f"テーブル: {table}")
print(f"衝突回数: {collisions}")

ヒント: 各名前の先頭文字の ord() を計算し（A=65, B=66, N=78, T=84）、% 7 でハッシュ値を求めてから追いましょう。
