説明: 4つのデータ構造

Pythonには「データをまとめて扱う」ための仕組みがいくつかあります。
この授業では、代表的な4つのデータ構造を学び、それぞれの得意・不得意を理解します。

1.1 リスト (list) -- 買い物リストのイメージ

買い物リストには順番があり、同じものが入ってもOK。後から追加・削除・並べ替えができます。 
Python
# リストは角括弧 [] で作る
shopping_list = ["りんご", "牛乳", "パン", "卵"]

# 要素を追加する（末尾に追加される）
shopping_list.append("バター")

# インデックス（番号）でアクセスする（0から始まる！）
print(shopping_list[0])   # "りんご"
print(shopping_list[2])   # "パン"

# リストは重複OK
shopping_list.append("りんご")
print(shopping_list)
# ['りんご', '牛乳', 'パン', '卵', 'バター', 'りんご']

1.2 辞書 (dict) -- 電話帳のイメージ

電話帳では「名前」で検索して「電話番号」を引きます。辞書はキー（名前）と値（番号）のペアでデータを管理します。 
Python
# 辞書は波括弧 {} で作る
phone_book = {
    "田中": "090-1111-2222",
    "鈴木": "080-3333-4444",
    "佐藤": "070-5555-6666",
}

# キーで値を取得する
print(phone_book["田中"])  # "090-1111-2222"

# 新しい人を追加する
phone_book["山田"] = "090-7777-8888"

# キーが存在するかチェックする
print("田中" in phone_book)  # True

1.3 タプル (tuple) -- 座標のイメージ

地図の座標 (35.6, 139.7) は「変えてはいけない」データです。タプルは作った後に変更できないリストです。 
Python
# タプルは丸括弧 () で作る
tokyo_location = (35.6762, 139.6503)

# インデックスでアクセスできる（リストと同じ）
print(tokyo_location[0])  # 35.6762

# タプルは変更できない（エラーになる！）
# tokyo_location[0] = 0  ← TypeError

# 複数の値を一度に変数に入れる（アンパック）
latitude, longitude = tokyo_location

# 関数から複数の値を返すときに便利
def get_min_max(numbers):
    return (min(numbers), max(numbers))

minimum, maximum = get_min_max([3, 1, 4, 1, 5, 9])

1.4 セット (set) -- 部員名簿のイメージ

部活の名簿には同じ人が2回入ることはありません。セットは重複を自動で排除するデータ構造です。 
Python
# セットは波括弧 {} で作る
members = {"田中", "鈴木", "佐藤", "田中"}
print(members)  # {'鈴木', '佐藤', '田中'} (田中は1回だけ)

# 要素が含まれるかチェック（超高速！）
print("田中" in members)  # True

# 集合演算ができる
tennis = {"田中", "佐藤", "高橋"}
soccer = {"鈴木", "佐藤", "山田"}

both = tennis & soccer     # 共通部分: {'佐藤'}
either = tennis | soccer  # 和集合: 全員

速度の違い -- どれを使うべき？

操作listdictsettuple
要素の検索 (in)遅い (順に探す)速い速い遅い
要素の追加速い (末尾)速い速い不可
順番の保持ありあり (3.7+)なしあり
重複許可キー不可不可許可
変更可能可能可能不可
  • 順番が大事でデータを後から変えたい → list
  • 名前（キー）でデータを引きたい → dict
  • 重複を排除したい、含まれるか高速に調べたい → set
  • 変更してはいけないデータ → tuple

速度の違いを体感する

Python
import time

# 大きなリストとセットを作る（0〜999999の数）
big_list = list(range(1_000_000))
big_set = set(range(1_000_000))

# リスト版（先頭から順に探すので遅い）
start_time = time.time()
result_list = 999_999 in big_list
list_time = time.time() - start_time
print(f"リストで検索: {list_time:.6f}秒")

# セット版（ハッシュで即座に見つけるので速い）
start_time = time.time()
result_set = 999_999 in big_set
set_time = time.time() - start_time
print(f"セットで検索: {set_time:.6f}秒")

リストのソートと検索

リストのソートとデータ検索の基本パターンを学びます。

Python
# --- ソートの基本 ---
scores = [78, 92, 65, 88, 71, 95, 83]
ascending = sorted(scores)

by_score = sorted(students, key=lambda x: x[1], reverse=True)
for rank, (name, score) in enumerate(by_score, start=1):
    print(f"  {rank}位: {name} - {score}点")
例題 5

ファイル入出力（CSV）

CSVファイルへの書き込みと読み込みの基本を学びます。

Python
# --- CSVファイルに書き込む ---
students = [("田中", 85), ("鈴木", 92), ("佐藤", 78)]
filename = "students.csv"

with open(filename, "w", encoding="utf-8") as f:
    f.write("名前,点数\n")       # ヘッダー行
    for name, score in students:
        f.write(f"{name},{score}\n")

# --- CSVファイルから読み込む ---
loaded = []
with open(filename, "r", encoding="utf-8") as f:
    lines = f.readlines()
    for i, line in enumerate(lines):
        if i == 0:
            continue  # ヘッダーをスキップ
        parts = line.strip().split(",")
        loaded.append((parts[0], int(parts[1])))

print(f"読み込み: {loaded}")
# [('田中', 85), ('鈴木', 92), ('佐藤', 78)]

模範コード（AL02-4.py）:

# ===== 発展課題: データ構造の変換と応用 =====
# CSVデータから科目ごとの平均点を辞書で計算する

# --- CSVデータ（文字列） ---
csv_data = """名前,科目,点数
田中,数学,85
鈴木,英語,92
田中,英語,78
鈴木,数学,88
佐藤,数学,76"""

# --- CSVを1行ずつ処理 ---
lines = csv_data.strip().split("\n")  # 改行で分割
header = lines[0]  # 1行目はヘッダー（使わない）

# 科目ごとの点数を集めるための辞書
# キー: 科目名, 値: 点数のリスト
scores = {}

for line in lines[1:]:  # 2行目以降を処理
    parts = line.split(",")  # カンマで分割
    name = parts[0]          # 名前
    subject = parts[1]       # 科目
    score = int(parts[2])    # 点数（整数に変換）

    # 科目がまだ辞書にない場合、空リストを作成
    if subject not in scores:
        scores[subject] = []

    # 点数をリストに追加
    scores[subject].append(score)

# --- 科目ごとの平均点を計算 ---
print("=== 科目ごとの平均点 ===")
for subject, score_list in scores.items():
    total = sum(score_list)          # 合計
    count = len(score_list)          # 人数
    average = total / count          # 平均
    print(f"{subject}: {average:.1f}点（{count}人）")

評価ポイント（これが説明できていればOK）:

  • CSVデータを split(",") で分割して各フィールド（名前・科目・点数）を取り出している
  • 辞書のキーに「科目名」、値に「点数のリスト」を使って、科目ごとにデータを整理している
  • sum() と len() を使って平均を計算する流れが説明できる
  • 辞書を使うことで、科目が増えても自動的に対応できる柔軟な設計になっている
