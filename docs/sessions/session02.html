説明: 4つのデータ構造

Pythonには「データをまとめて扱う」ための仕組みがいくつかあります。
この授業では、代表的な4つのデータ構造を学び、それぞれの得意・不得意を理解します。

1.1 リスト (list) -- 買い物リストのイメージ

買い物リストには順番があり、同じものが入ってもOK。後から追加・削除・並べ替えができます。 
Python
# リストは角括弧 [] で作る
shopping_list = ["りんご", "牛乳", "パン", "卵"]

# 要素を追加する（末尾に追加される）
shopping_list.append("バター")

# インデックス（番号）でアクセスする（0から始まる！）
print(shopping_list[0])   # "りんご"
print(shopping_list[2])   # "パン"

# リストは重複OK
shopping_list.append("りんご")
print(shopping_list)
# ['りんご', '牛乳', 'パン', '卵', 'バター', 'りんご']

1.2 辞書 (dict) -- 電話帳のイメージ

電話帳では「名前」で検索して「電話番号」を引きます。辞書はキー（名前）と値（番号）のペアでデータを管理します。 
Python
# 辞書は波括弧 {} で作る
phone_book = {
    "田中": "090-1111-2222",
    "鈴木": "080-3333-4444",
    "佐藤": "070-5555-6666",
}

# キーで値を取得する
print(phone_book["田中"])  # "090-1111-2222"

# 新しい人を追加する
phone_book["山田"] = "090-7777-8888"

# キーが存在するかチェックする
print("田中" in phone_book)  # True

1.3 タプル (tuple) -- 座標のイメージ

地図の座標 (35.6, 139.7) は「変えてはいけない」データです。タプルは作った後に変更できないリストです。 
Python
# タプルは丸括弧 () で作る
tokyo_location = (35.6762, 139.6503)

# インデックスでアクセスできる（リストと同じ）
print(tokyo_location[0])  # 35.6762

# タプルは変更できない（エラーになる！）
# tokyo_location[0] = 0  ← TypeError

# 複数の値を一度に変数に入れる（アンパック）
latitude, longitude = tokyo_location

# 関数から複数の値を返すときに便利
def get_min_max(numbers):
    return (min(numbers), max(numbers))

minimum, maximum = get_min_max([3, 1, 4, 1, 5, 9])

1.4 セット (set) -- 部員名簿のイメージ

部活の名簿には同じ人が2回入ることはありません。セットは重複を自動で排除するデータ構造です。 
Python
# セットは波括弧 {} で作る
members = {"半沢直樹", "伊佐山泰二", "黒崎駿一", "半沢直樹"}
print(members)  # {'伊佐山泰二', '黒崎駿一', '半沢直樹'} (半沢直樹は1回だけ)

# 要素が含まれるかチェック（超高速！）
print("半沢直樹" in members)  # True

# 集合演算ができる
tokyo_central = {"半沢直樹", "黒崎駿一", "大和田暁"}
osaka_nishi = {"伊佐山泰二", "黒崎駿一", "中野渡謙"}

both = tokyo_central & osaka_nishi     # 共通部分: {'黒崎駿一'}
either = tokyo_central | osaka_nishi  # 和集合: 全員

ハッシュ関数って何をしているの？

上の図で出てきた「ハッシュ関数」を、もう少し詳しく見てみましょう。

ハッシュ関数 = 「データを受け取って、番号を返す」計算のルール

みなさんの学籍番号を想像してみてください。
学校は学生の名前ではなく学籍番号で管理していますよね。
名前は被ることがあるし、探しづらい。でも番号なら「131124108番の棚を見ればいい」と一発で見つかる。

ハッシュ関数は、これと同じことをコンピュータ内部で自動的にやっています。
「データ（文字列や数値）を受け取り → 固定範囲の番号に変換する」のが仕事です。

ハッシュ関数の3つのポイント

  • 同じ入力 → 必ず同じ番号: 「佐藤」を何回入れても、必ず80番が返る。だから「80番を見に行けばいい」と分かる
  • 計算は一瞬: 足し算と割り算だけなので、データが100件でも10万件でも計算時間はほぼ同じ
  • 余りを使うのがコツ: % 100 で余りを求めると、どんなに大きな数でも 0〜99 の範囲に収まる。これがテーブルの「棚番号」になる

※ 実際のPythonの辞書やセットはもっと高度なハッシュ関数を使っていますが、基本の考え方はここで説明した通りです。 ハッシュ関数の詳細は第6回・第7回の「ハッシュ法」で本格的に学びます。

速度の違い -- どれを使うべき？

操作listdictsettuple
要素の検索 (in)遅い (順に探す)速い速い遅い
要素の追加速い (末尾)速い速い不可
順番の保持ありあり (3.7+)なしあり
重複許可キー不可不可許可
変更可能可能可能不可
  • 順番が大事でデータを後から変えたい → list
  • 名前（キー）でデータを引きたい → dict
  • 重複を排除したい、含まれるか高速に調べたい → set
  • 変更してはいけないデータ → tuple

リストのソートと検索

リストのソートとデータ検索の基本パターンを学びます。

Python
# --- ソートの基本 ---
scores = [78, 92, 65, 88, 71, 95, 83]
ascending = sorted(scores)
print(f"昇順: {ascending}")   # [65, 71, 78, 83, 88, 92, 95]

descending = sorted(scores, reverse=True)
print(f"降順: {descending}")  # [95, 92, 88, 83, 78, 71, 65]

# --- タプルのリストをソートする ---
students = [("田中", 78), ("鈴木", 92), ("佐藤", 65)]

# 点数でソートする（key=lambda で指定）
by_score = sorted(students, key=lambda x: x[1], reverse=True)
for rank, (name, score) in enumerate(by_score, start=1):
    print(f"  {rank}位: {name} - {score}点")

→ 並び順はどう変わった？ x[1] は点数、x[0] は名前。何を基準にソートするかが変わる。

例題 5

ファイル入出力（CSV）

CSVファイルへの書き込みと読み込みの基本を学びます。

Python
# --- CSVファイルに書き込む ---
students = [("田中", 85), ("鈴木", 92), ("佐藤", 78)]
filename = "students.csv"

with open(filename, "w", encoding="utf-8") as f:
    f.write("名前,点数\n")       # ヘッダー行
    for name, score in students:
        f.write(f"{name},{score}\n")

# --- CSVファイルから読み込む ---
loaded = []
with open(filename, "r", encoding="utf-8") as f:
    lines = f.readlines()
    for i, line in enumerate(lines):
        if i == 0:
            continue  # ヘッダーをスキップ
        parts = line.strip().split(",")
        loaded.append((parts[0], int(parts[1])))

print(f"読み込み: {loaded}")
# [('田中', 85), ('鈴木', 92), ('佐藤', 78)]

リスト vs セットの検索速度を体感する ── AL02-1.py

100万件のデータの中から1つの値を探す速さを、リストとセットで比較します。
説明セクションで学んだ「ハッシュ関数」の威力を、実際の数値で確認しましょう。

手順

  1. VS Code を開き、新規ファイルを作成する（メニュー → File → New File）
  2. ファイル名を AL02-1.py にして保存する（Ctrl+S / Cmd+S）
  3. 下のコードの「コピー」ボタンをクリックして、VS Code に貼り付ける（Ctrl+V / Cmd+V）
  4. ターミナルで実行する（右上の ▶ ボタン、または python AL02-1.py）
  5. 表示された秒数を見比べる
Python ── AL02-1.py
import time

# ===== 準備: 100万件のデータを用意する =====
big_list = list(range(1_000_000))   # リスト版
big_set  = set(range(1_000_000))    # セット版

# ===== リストで探す（先頭から順番に） =====
start = time.time()
result = 999_999 in big_list
list_time = time.time() - start
print(f"リストで検索: {list_time:.6f}秒")

# ===== セットで探す（ハッシュで一発） =====
start = time.time()
result = 999_999 in big_set
set_time = time.time() - start
print(f"セットで検索: {set_time:.6f}秒")

# ===== 何倍違うか計算 =====
if set_time > 0:
    print(f"→ リストはセットの約{list_time / set_time:.0f}倍遅い")
else:
    print("→ セットは速すぎて計測不能！")

実行結果の例

リストで検索: 0.012345秒
セットで検索: 0.000001秒
→ リストはセットの約12345倍遅い

※ 数値はPCの性能により異なりますが、数千〜数万倍の差が出るはずです。

なぜこんなに差が出る？ リストは100万件を1番目から順番に見ていく。セットはハッシュ関数で「置き場所」を計算して一発で見つける。データが増えるほど差は広がります。

標準課題 1

余り（%）で値を範囲に収めるトレース

授業で学んだハッシュ関数は、%（余り）を使って大きな数を一定の範囲に収めていました。
以下のコードを実行せずに、各学生がどのグループに振り分けられるかを予測しなさい。
予測を書いた後に、AL02-2.py として保存・実行して確かめなさい。

Python ── AL02-2.py
students = [
    ("半沢直樹", 131124108),
    ("大和田暁", 131324239),
    ("黒崎駿一", 132200015),
    ("伊佐山泰二", 131100042),
    ("中野渡謙", 132400077),
]

group_count = 3  # 3グループに分ける

for name, student_id in students:
    group = student_id % group_count
    print(f"{name}（{student_id}）→ グループ{group}")

ヒント: % は「割り算の余り」です。131124108 % 3 は「131124108 を 3 で割った余り」。余りは必ず 0, 1, 2 のどれかになります。

リスト＋辞書＋ループのトレース

以下のコードを実行せずに、ループの各ステップで scores と result がどう変化するかを追い、最終出力を予測しなさい。
予測を書いた後に、AL02-3.py として保存・実行して確かめなさい。

Python ── AL02-3.py
students = [("田中", 80), ("鈴木", 65), ("佐藤", 90), ("山田", 70)]
scores = {}
result = []

for name, point in students:
    if point >= 75:
        scores[name] = "合格"
        result.append(name)
    else:
        scores[name] = "不合格"

print("scores =", scores)
print("合格者:", result)
print("合格者数:", len(result))

ヒント: ループの各回で (name, point) がどの値になるか、1行ずつ追いましょう。scores 辞書と result リストの両方を更新していることに注意。
